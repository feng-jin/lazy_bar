# Lazy Bar

Lazy Bar 是一个面向上班族的 macOS 菜单栏盯盘工具，目标是在不打扰正常工作的前提下，把关键行情信息持续放在 bar 上。

## 当前状态
- 当前仓库处于以真实行情为主的早期可用阶段。
- 当前是单 target 的 macOS 原生应用；菜单栏入口由 AppKit 承载，主面板内容继续复用 SwiftUI 视图。
- 当前默认通过新浪财经未公开快照接口拉取 A 股准实时行情，并按 A 股交易时段动态轮询刷新：交易时段内每 3 秒拉取一次，非交易时段每 10 分钟拉取一次；仓库仍保留 mock 行情 provider 供 Preview 和后续兜底使用。watchlist 不再依赖项目内 JSON 资源，首次启动默认为空列表，由用户在设置页逐条手动录入并持久化。
- 首次启动且 watchlist 为空时，菜单栏会直接显示“请先添加股票”作为空态提示，避免状态栏 item 因无数据而看起来像没有启动。
- 当前菜单栏可展示股票代码、股票简称、最新价和涨跌幅，展示字段可在独立设置弹窗中通过带说明的两列卡片式选项勾选；设置弹窗已拆成“监控股票”和“展示字段”两个 tab，其中“监控股票”tab 收敛为单段式编辑区：顶部一行录入简称与代码，下面紧接固定高度的可编辑列表，支持手动新增、删除和直接编辑股票代码/简称；展示字段则单独放在第二个 tab 中，并要求至少保留一个字段可见，避免 bar 和主面板进入空展示状态。左键主面板中的股票区域在达到上限后会滚动，避免占用过大空间。点击保存会立即提交但不关闭弹窗，点击取消会放弃本轮未保存修改并关闭弹窗；如果这次保存只改了展示字段或股票简称、没有改 symbol 列表，bar 和左键主面板会直接基于现有行情快照重算，不再等下一次拉行情；如果改了 symbol 列表，则会先保留当前仍有效的已加载行情并立即后台拉取新列表，避免 bar 退回成启动时的“加载中...”空态。bar ticker 在展示数据或顺序变化时会按新列表重启动画，确保与左键列表按同一份最新数据同步切换。
- 左键点击菜单栏会从状态栏图标下方展开无箭头的主面板；面板分成上下两个区域，上半部分是股票列表，下半部分只保留设置、退出以及后续少量功能入口。底部操作区使用更接近系统下拉菜单项的整行点击、高亮反馈与图标样式。面板整体使用更贴近 macOS 原生菜单的材质、圆角边界、分隔线与 hover 高亮，并优先与 bar 使用同一宽度和左右边界对齐；bar 与上半部分股票列表会共享同一份 `MenuBarViewModel.renderState` 衍生出的 presentation、同一套列结构、主副文字体风格、数值字体风格，以及按当前股票里最长字段动态计算出的列宽。`StatusBarController` 现在只负责把这份 presentation 同步到状态栏宿主与已打开的主面板容器尺寸，避免 bar、popover 和内部股票列表虽然读的是同一份数据，但外层宽高语义短暂分裂；当前股票简称列、股票代码列、股价列、涨跌幅列统一采用右对齐，尽量让字段尾部边界更整齐，其中身份列到股价列之间会保留比其他列更宽的基准间距，避免股价位数变化时名称与股价挤得过近；当关闭部分字段后，bar 和主面板宽度也会随当前可见列一起收紧，保持紧凑展示。
- 设置弹窗由 AppKit 独立窗口承载，设置内容继续复用 SwiftUI `SettingsView`；左键面板本身不再承载具体设置表单。
- 当前没有第三方依赖，也没有 Swift Package 依赖，工程只使用系统框架。
- 产品方向与详细边界见 [docs/product.md](docs/product.md)。

## 代码结构
- `Sources/Models`：领域模型与展示模型。
- `Sources/Providers`：数据来源边界与 provider 实现。
- `Sources/ViewModels`：菜单栏与设置窗口的状态协调，以及 `QuoteSession` 这类窄职责上层协调对象。
- `Sources/Views`：SwiftUI 视图，以及菜单栏共享样式、纯展示 `MenuBarPresentation`、行组件、列布局计算等展示层基础设施。
- `Sources/App`：AppKit 壳层、状态栏宿主/主面板协调与设置持久化。
- `Sources/PreviewSupport`：Preview 所需的示例数据。
- `Resources`：`Info.plist` 与资源文件。

## 开发前提
- 目标平台：macOS。
- 最低系统版本：macOS 15.0。
- UI 技术栈：SwiftUI 优先，仅保留系统框架范围内的能力。
- 建议使用完整 Xcode 环境；仅安装 Command Line Tools 时，`xcodebuild` 无法完成完整工程构建。
