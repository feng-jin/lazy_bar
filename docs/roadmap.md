# Roadmap

## Phase 1: 完成 V1 菜单栏盯盘
目标是完成第一版最小可用能力：先稳定单只个股的菜单栏盯盘体验，再演进到多只轮播。

新增或完善的重点：
- 明确菜单栏最小信息集合，默认保留股票简称、最新价和涨跌幅，并允许用户在设置里勾选股票代码等字段。
- 提供点击后的极简操作菜单，只保留设置和退出。
- 提供最小设置能力，支持菜单栏展示字段和颜色开关配置。
- 明确 loading、不可用、空态等基础状态。
- 保持 mock 数据链路稳定，方便后续替换真实 provider。

这一阶段暂不做：
- 真实行情接入。
- 复杂详情面板。
- watchlist 配置。
- 大而全的高级自定义项。

## Phase 2: 接入真实 A 股行情
目标是让应用具备最基础的真实盯盘能力，但仍保持轻量。

新增或完善的重点：
- 新增真实行情 provider。
- 明确请求失败、超时、不可用时的降级策略。
- 将 mock/real provider 切换保持在依赖注入层完成。

这一阶段暂不做：
- 复杂技术指标。
- 重型图表。
- 下单或交易相关能力。

## Phase 3: 支持多标的轮播
目标是把第一版轮播从“能用”推进到“稳定、清晰、可持续使用”。

新增或完善的重点：
- 优化多标的轮播策略，确保切换节奏和可读性平衡。
- 扩展数据模型与 ViewModel，使其更好支持多标的状态。
- 优化长代码名、异常值、停牌或缺失数据时的展示行为。

这一阶段暂不做：
- 复杂拖拽排序。
- 大量高级自定义项。

## Phase 4: 刷新策略与交易时段逻辑
目标是把“准实时刷新”从口号变成稳定行为。

新增或完善的重点：
- 定义刷新频率与触发方式。
- 区分交易时段与非交易时段的行为。
- 处理网络异常、频率控制与资源消耗问题。

这一阶段暂不做：
- 推送式实时流订阅。
- 以高频交易需求为目标的超低延迟优化。

## Phase 5: Watchlist 配置
目标是让用户可以逐步把工具变成自己的常用盯盘入口。

新增或完善的重点：
- 支持维护关注标的列表。
- 定义默认排序、展示优先级和设置入口形态。
- 保持配置能力克制，不破坏“低打扰高隐蔽”的产品原则。

这一阶段暂不做：
- 大而全的设置中心。
- 复杂同步体系。

## Phase 6: 补充详情与扩展信息
目标是在不破坏“低打扰高隐蔽”原则的前提下，谨慎增加二级信息。

新增或完善的重点：
- 评估是否需要轻量详情面板。
- 如果增加详情，继续以补充说明为主，不让其反客为主。
- 控制信息层级，避免演化成重型行情终端。

这一阶段暂不做：
- 复杂图表工作台。
- 与核心盯盘无关的信息堆叠。

## Phase 7: 错误态与降级体验完善
目标是在真实使用场景下保证体验可靠。

新增或完善的重点：
- 完善错误态、空态、过期数据提示。
- 优化极端场景下的显示策略。
- 补齐必要的可观测性和问题定位信息。

这一阶段暂不做：
- 与产品核心价值无关的装饰性功能堆叠。
